        WebPBCall is a 100% Java set of PowerJ classes and Powerbuilder non-visual-object (NVO) that enables Distributed Powerbuilder (DPB) objects to be accessed as part of a Java
WebPBCall, along with the Powerbuilder WEB.PB option, brings the familiarity and power of Powerbuilder database access and PowerScript programming to Java applet/applications for the Internet or Intranet. Acting as a client-interface to WEB.PB, WebPBCall allows DPB user-object functions, with or without parameters, to be called to produce a result set, or update tables from Java applets/applications.

This ability allows developers to use existing Powerbuilder DataWindow objects as a datasource to any Java applet/application without using JDBC client software/drivers or any other middleware. The only software required is WEB.PB installed on the web server.

In the simplest terms, WebPBCall creates a URL to invoke a Powerbuilder server object, which in turn creates a result-set of data from a DataWindow or datastore, along with some column metadata information. On the Java side, the WebPBCall Java classes parse the result-set . At this point the data is accessible via JDBC ResultSet Interface, JDBC ResultSetMetaData Interface, and other low-level data access methods.

The Powerbuilder server object and WebPBCall use the WEB.PB CGI program as a conduit to invoke methods and move data.
 

The Pieces

WebPBCall PowerJ Java classes:

WebPBCall - responsible for establishing a connection to WEB.PB, retrieving data and creating the WebPBResultSet object.

 WebPBResultSet - responsible for implementing the ResultSet interface and converting the string only result-set to Java objects and primitive data types.

 WebPBResultSetMetadata - responsible for returning JDBC and SQL metadata information for columns in the result set.

Powerbuilder object:

n_cst_java_metadata - Powerbuilder NVO for creating ResultSet Metadata. This NVO builds a string that contains both the result-set and metadata data. A developer creates an instance of this object and then calls any of the following methods:

public function string of_getmetadata (datastore as_ds, string as_result) - This method will return a result set that is sent to WebPBCall for parsing.

Datastore - Datastore object used to produce a result-set
String - the data produced from the Datastore

Input parameters specify the Datastore object and the string data result from the Datastore is used to produce the final result set; it parses the Datastore to extract column metadata and then combines this with the the string data result.

public function string of_executesql (string as_sql, ref transaction as_trans) - This method will execute a SQL statement and return the result set for the statement.

Notes 

Distributed Powerbuilder

User NVO objects:

DPB functions should return data as a tab-delimited string. This string can be easily produced using a Datastore and a DataWindow object. After the Datastore retrieves the data, and after any data manipulation, the Describe PowerScript function is used to produce the tab-delimited string, example:
 datastore.desrcibe("datawindow.data")
 DPB functions can be used to produce result sets, or they can be used to update a database table.

To retrieve data, PDB object can use a Datastore, dynamic SQL or anything else that produces a result set. The result set must be a string that separates each column with a "\t" character, and each row with a "\r\n."

Note: If data is produced outside the scope of a Datastore/DataWindow object, the Java JDBC ResuletSetMetaData will not be supported.

n_cst_java_metadata:

To create the final result set to pass back to the WebPBCall Java classes, the function of_getmetadata() in n_cst_java_metadata is called. This function takes the dataobject and the actual data as arguments. This object pareses the passed in dataobject in order to create the metadata information.

Example:

n_cst_java_metadata invo_mdata
transaction itr_trans
datastore lds_order
string ls_result=""
// create datastore and set DataWindow object
lds_order = create datastore
lds_order.dataobject = "d_sale_orders"
// attach datastore to transaction object
lds_order.settransobject(itr_trans)
// retrieve data and extract tab-delimited format
if (lds_order.retrieve() >0) then
    ls_result = lds_order.Describe("datawindow.data")
end if
// call function to create final result set for WebPBCall
ls_result = invo_mdata.of_getmetadata(lds_order, ls_result)
// clean up
destroy lds_order
// return final resultset to WebPBCall
return ls_result
 

WebPBCall:

Current restrictions inherent in Web.PB restrict result sets to type string and blob. WebPBCall only supports string type. To overcome this imitation, WebPBCall provides getItemXXX function that covert string type to the specified datatype.
WebPBCall uses the Java IO, Net and Util packages to connect and communicate with WEB.PB. This communication allows a DPB object to use existing DataWindows, and execute dynamic SQL statements, to return a result set, or update tables.
Multiple arguments passed to DPB function should be separated by a ' character.
 
WebPBCall Dataflow
The Java applet/application uses WebPBCall as follows:
      1. Construct the WebPBCall class object
      2. Set parameters, using object methods, to specify the pieces that make up the URL needed to invoke CGI call to WEB.PB
      3. Call the retrieve() method to retrieve data from DPB. The data is stored in WebPBCall.
      4. The applet/application can now call any of the WebPBCall methods to get the data, attach a query object to get the data, create a ResultSet object and use ResultSet interface methods to get the data, or get the ResultSetMetaData object and get data column information.

 Setting up the sample

In order for the sample to work correctly, the above files and the following steps are required:

Setup:

ODBC:

Create an ODBC data source "nidemodb2" to point to the nidemodb2.db database.

WEB.PB:

See Internet Developer Toolkit (IDT) for list of files and system configuration. The only changes required for the sample is in the pbweb.ini file. Add the following: [niexdpbs]
application="niexdpbs"
// change to reflect the server IP address.
location="251.251.251.0"
serverobject=n_cst_java
driver="winsock"
options="MaxRetry=200"
  The "services" files should contain the entry:
(server application name port/protocol): niexdpbs 10100/tcp

Webserver:

Set Web server mapping to point to the directory containing index.html and WebPBSampleZip.zip

This applet uses a web site to access the WebPB CGI program and image files. You can use the web server shipped with Powerbuilder IDT, or you can use any web server. Please refer to IDT for configuration options. This sample uses the web server that ships with IDT. The main difference between the various web servers is the CGI program you will invoke. For example, for O'Reilly & Associates WebSite Server, invoke PBCGI050.EXE. For Microsoft Information Server, invoke PBISA050.DLL.

Execution:

    Run Niexample.exe and click Start Server - (if you get an error, try the following Troubleshooter section)

    Run browser and enter URL set above

    Click "Retrieve Employees" button
     

Example using O'Reilly & Associates WebSite Server:

Create directory: C:\website\htdocs\webpbdemo

Open the Properties for WebSite (right-mouse on Website Icon on Toolbars)

Select Mapping tab. Add "/WebPBDemo" in Document URL Path, and C:\website\htdocs\webpbdemo to Directory (full or document-relative), and click Add.

Copy webpbcalldemo.html and WebPBSampleZip.zip to C:\website\htdocs\webpbdemo

Run browser and enter the URL: http://localhost/webpbdemo/webpbcalldemo.html

Troubleshooter 

Make sure there is an entry in registry file for niexample.exe. The entry should contain a path to the Powerbuilder run time files, and the directory of niexample.exe.

Make sure niexample.exe, webpb_resultset.pbd and niexdpbs.pbd are in the same directory.

Make sure there is an " Nidemodb2" ODBC entry that points to the nidemodb2.db.

The "services" files should contain the entry:
(server application name port/protocol):
niexdpbs 10100/tcp
